## Install, verify, and run

```bash frame="none"
# Linux or macOS. Auto-detects CUDA, Metal, or MKL.
curl --proto '=https' --tlsv1.2 -sSf https://raw.githubusercontent.com/EricLBuehler/mistral.rs/master/install.sh | sh

# Windows, from PowerShell.
irm https://raw.githubusercontent.com/EricLBuehler/mistral.rs/master/install.ps1 | iex
```

```bash frame="none"
# Check detected hardware, build features, and Hugging Face connectivity.
mistralrs doctor

# Chat with a model in your terminal.
mistralrs run --quant 4 -m Qwen/Qwen3-4B

# Or serve the same model behind an OpenAI-compatible HTTP API with a built-in web UI.
mistralrs serve --quant 4 -m Qwen/Qwen3-4B
```

## Features

<CardGrid>
  <Card title="Auto-detection" icon="rocket">
    `mistralrs run -m <model>` works without flags. The binary infers the
    architecture, chat template, and accelerator from the Hugging Face
    repository.
  </Card>
  <Card title="Quantization" icon="puzzle">
    `--quant 4` prefers a prebuilt UQFF from `mistralrs-community` if one is
    published, otherwise applies in-situ quantization. `--quant auto` picks
    for your hardware. Supported methods: ISQ, GGUF, GPTQ, AWQ, HQQ, AFQ, FP8,
    and MXFP4.
  </Card>
  <Card title="Auto-tuning" icon="seti:config">
    `mistralrs tune -m <model>` benchmarks the host and prints recommended
    settings for context length, batch size, and quantization level.
  </Card>
  <Card title="Diagnostics" icon="magnifying-glass">
    `mistralrs doctor` reports detected hardware, compiled accelerator
    features, and Hugging Face connectivity before you load a model.
  </Card>
  <Card title="Agents" icon="open-book">
    Local agent runtime with a server-side tool loop, web search, code
    execution, MCP, generated media, and sessions.
  </Card>
  <Card title="Multi-GPU" icon="seti:hex">
    Tensor parallelism across local GPUs via NCCL, or across machines via the
    built-in ring backend.
  </Card>
  <Card title="Web UI" icon="laptop">
    Mounted at `/ui` by default. Browser chat with reasoning blocks, tool and
    code-execution call visualization, and inline search results. Pass `--no-ui`
    to disable.
  </Card>
</CardGrid>
